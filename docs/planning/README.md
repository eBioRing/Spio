# Planning Docs

**Purpose:** Hold delivery phases, workstreams, and planning summaries for `spio`.

**Last updated:** 2026-04-17

## Scope

- master plan
- ecosystem delivery mirror
- ecosystem file-governance alignment mirror
- workstreams
- bootstrap summary
- stage reviews and retrospective summaries
- future-direction and cross-team coordination summaries

## Maintenance Rule

Planning documents describe what to do and in what order. They do not own policy or gate command definitions.

When a milestone or checkpoint spans all three repos, update the local mirror first:

1. `Styio-Ecosystem-Delivery-Master-Plan.md` for product milestones and repo exits
2. `Styio-Ecosystem-File-Governance-Alignment-Plan.md` for docs tree, file governance, ignore-policy, and repo hygiene alignment

Repo-only sequencing then belongs in `Spio-Master-Plan.md` and workstream files.
